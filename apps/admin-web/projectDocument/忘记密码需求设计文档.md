# 一、项目技术栈

* 前端：Vue3 + JavaScript
* 后端：腾讯云函数
* 数据库：腾讯云开发数据库 / uniCloud 风格调用

* 密码存储：使用 sha256 哈希存储
* 代码风格：清晰、可维护、加必要注释，尽量不要引入不必要依赖

# 二、我要实现的业务功能

实现“忘记密码”功能，流程如下：

1. 用户在登录页点击“忘记密码”
2. 用户输入账号（账号可以是 username 或 email）
3. 后端根据账号查找用户
4. 如果账号存在且绑定邮箱，则向该注册邮箱发送 6 位验证码
5. 前端输入验证码
6. 后端校验验证码是否正确、是否过期、是否已使用
7. 验证通过后，后端生成一个 resetToken 返回给前端
8. 前端输入新密码和确认密码
9. 前端调用重置密码接口，提交 resetToken + newPassword
10. 后端校验 resetToken，有效则更新用户密码
11. 更新成功后，本次验证码记录失效，流程结束

注意：

* 邮箱验证码不是登录，而是用于“找回密码”
* 不允许用户仅凭验证码直接登录
* 必须分为“验证码验证通过”和“真正修改密码”两个步骤

# 三、安全要求

严格按下面要求实现：

1. 验证码为 6 位数字
2. 验证码有效期 5 分钟
3. 验证码只能使用一次
4. 数据库里不能存明文验证码，只能存 codeHash
5. 忘记密码发送验证码接口，不要暴露“账号是否存在”

   * 无论账号是否存在，接口都统一返回：
   * “如果账号存在，验证码已发送到绑定邮箱”
6. 新密码不能明文存储，必须 sha256 哈希后保存
7. resetToken 必须是随机字符串，并且只能用于本次重置密码
8. 所有 SMTP 配置、JWT/密钥类配置必须使用环境变量读取
9. 代码中不要硬编码邮箱账号、授权码、密钥
10. 要加基础限流逻辑注释或预留位置，至少提示：

* 同一账号 60 秒内只能发送一次验证码
* 同一账号每天限制发送次数
* 同一 IP 要限制频率

11. 接口错误提示尽量统一，不要泄露过多内部信息
12. 新密码至少 8 位，后端也要校验

# 五、需要实现的接口 / 云函数

请不要拆成多个云函数。
请统一实现为 **一个腾讯云函数**：

## 云函数名称

`forgetPasswordService`

## 调用方式

通过传入不同的 `action` 值来区分具体功能。

支持以下 3 个 action：

### 1）action = "sendResetCode"

作用：

* 接收参数：`action`, `account`
* 根据 username 或 email 查找用户
* 如果存在且有绑定邮箱，则生成验证码并发送邮件
* 在 `password_reset_codes` 中保存验证码哈希和有效期
* 无论账号是否存在，最终都统一返回成功提示

请求参数示例：

```json
{
  "action": "sendResetCode",
  "account": "admin"
}
```

返回格式示例：

```json
{
  "code": 0,
  "message": "如果账号存在，验证码已发送到绑定邮箱"
}
```

---

### 2）action = "verifyResetCode"

作用：

* 接收参数：`action`, `account`, `code`
* 查找该账号最新一条未使用的 forgot_password 验证码记录
* 校验验证码、是否过期
* 成功后生成 `resetToken`
* 更新该记录：

  * `verified = true`
  * `verifyToken = xxx`
  * `verifiedAt = 当前时间`
* 返回 `resetToken`

请求参数示例：

```json
{
  "action": "verifyResetCode",
  "account": "admin",
  "code": "123456"
}
```

成功返回格式：

```json
{
  "code": 0,
  "message": "验证成功",
  "resetToken": "xxxxx"
}
```

失败时统一返回：

```json
{
  "code": 401,
  "message": "验证码错误或已失效"
}
```

---

### 3）action = "resetPassword"

作用：

* 接收参数：`action`, `resetToken`, `newPassword`
* 校验 `resetToken` 是否存在、是否已验证、是否未使用、是否未过期、`scene` 是否为 `forgot_password`
* 校验新密码长度至少 8 位
* 更新 `users` 表中的 `passwordHash`
* 将当前验证码记录标记为：

  * `used = true`
  * `usedAt = 当前时间`
* 返回修改成功

请求参数示例：

```json
{
  "action": "resetPassword",
  "resetToken": "xxxxx",
  "newPassword": "abc123456"
}
```

返回格式示例：

```json
{
  "code": 0,
  "message": "密码修改成功"
}
```

---
